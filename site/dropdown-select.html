DropdownSelect.js

Use DropdownSelect.js to transform a standard HTML <select> element into a Bootstrap-styled dropdown with checkbox or radio-button options. The component preserves standard form behavior while providing a flexible and user-friendly selection interface.

How it works

When a standard HTML <select> element is decorated as a DropdownSelect, it is re-rendered as a Bootstrap dropdown button with the original options displayed as either radio inputs (for single-select elements) or checkboxes (for multiple selects).

Options may define a data-submit-name attribute to submit their values using distinct name/value pairs instead of the parent select's name.

Summarize Multiple

By default, DropdownSelect will attempt to display selected option labels in the space containing the select (re-rendered as a dropdown button). If instead you would like to see a count summary (2 selected, 3 selected, etc.) after the first selection, set the summarizeMultiple option to true or use the data attribute data-summarize-multiple="true"

Notice that clicking an option with a long label, even if the first one, may still result in the summarized label if the label does not fit.

Clear / Select All

Use showClear or data-show-clear="true" to include a clear button to unselect all shown options.

Use showSelectAll or data-show-select-all="true" on multiple selects to include a select all button to select all shown options.

Filter Options

To add a filter input, set the showFilter option to true or add data-show-filter="true"

The default filter behavior will only hide options not matched by the filter, not unselect them. To unselected options excluded by the filter set the filterBehavior to unselect-hidden or data-filter-behavior="unselect-hidden". The default filterBehavior value is only-hidden.

Custom Submission Names

To submit an option using a unique parameter name, specify data-submit-name="newName" on the option element. This allows DropdownSelect to submit multiple distinct name/value pairs instead of using only the parent select's name.

Styled

An <option>'s class attribute is applied to a <span> wrapping the rendered item text, allowing text-level styling without affecting the dropdown item's layout or behavior. Inline style attributes are ignored.

Options

Name Type Default Description
summarizeMultiple boolean false If true, the label '# selected' is shown when multiple options are selected. Otherwise, the full option label (comma joined) will be shown if space permits.
showFilter boolean false If true, includes a filter input.
filterBehavior string 'only-hidden' Two values: only-hidden (default) and unselect-hidden.
showClear boolean false If true, includes a clear button.
showSelectAll boolean false If true, includes a select all button.

Properties

Property Description
value

Contains the names and values of all selected items. Normally, if not adding extra options with data-submit-name, value is an object having a single property, the name of the <select>, with an array of selectecd items.
{ 'order': [ 'pork' ] } or { 'order': [ 'pork', 'beef', 'fish' ] }

When setting the value, the same object form should be used or, when only a single item is selected, a convenience form can be used where the selected item is provided as a single string value.
{ 'order': 'pork' } or { 'order': [ 'pork' ] }

Methods

Method Description
syncFromSelect Allows external (programatic) changes to the DropdownSelect's source of true, the <select>, to resync item state based on the underlying <select> element. For example, if a user clicks a reset button, the browser resets native form elements but knows nothing about our enhanced select (checkbox/radio inputs). This method (syncFromSelect) lets you hook into that.
selectAll Select all shown select options. This has no effect if the underlying <select> is not multiple.
clear Unselects all shown select options.
destroy Removes all elements and event listeners created during initialization from the DOM.
