init-list-items.js

This module handles adding and removing list items along with updating counts.

How it works

Two click listeners are added to the document. The first will add an item to the list and the second will remove it. There is a few parts to setting this up detailed below.

Examples

Start by adding the initialization JavaScript to your page.

List Item Components

Here is a list of each component. Components are marked as optional or required.

Add Button or Link (Required)

Adds a new list item to the list container using a template.

Attribute Required Description
data-isp-toggle="add-item" true This attribute marks this element as an add button/link for the init-list-items.js.
data-template true This attribute is for a JavaScript selector that is used to select the template element with a list item template.
data-list true This attribute is for a JavaScript selector that is used to select the list container that the list item will be inserted into.

Remove Button or Link (Required)

Removed or hides a list item in a list.

Attribute Required Description
data-isp-toggle="remove-item" true This attribute marks this element as a remove button/link for the init-list-items.js.
data-parent true This attribute is for a JavaScript selector that is used to get the closest parent that is the list item itself.

Delete Input (Optional)

When present in a list item, will cause the list item to be hidden instead of deleted. The input value is set to true as well.

This input should only be added if item in the list needs to be removed server side. It should never be part of the template, as new items that are deleted right away do not need to be deleted server side.
Attribute Required Description
data-isp-toggle="remove-item" true This attribute marks this element as a remove button/link for the init-list-items.js.
data-delete-input true This attribute is to mark this input as a delete flag for the server. This is also a trigger that tells the module to not remove the list item from the DOM but to hide it instead.

Count Element (Optional)

When present in a list item, will have the inner HTML replaced with the item count.

Attribute Required Description
data-item-count true This attribute marks this element as a count element for the init-list-items.js.

Template (Required)

A template element containing a block of HTML representing a list item.

All the examples below will be in pure HTML. If you are using a templating engine to generate your HTML pages, use a fragment/tag[1] for the body of the template and for each list item in a loop. It will make things easier to maintain.

[1]: 'Fragment', 'Tag', 'Template', whatever a re-usable HTML block of code is called.
Attribute Required Description
data-index-var false Replaces the default search token ({index}) used to find where the index is inserted into the template.
data-insert-location false Specify if the new item should be inserted into the top or bottom of the list.

List Container (Required)

An element containing list item children. While the example is a list-group, it can be any element with children.

Do not add other elements in the list container. It will cause this module to report invalid counts.
Attribute Required Description
data-index false Specify the next index to use instead of calculating it.
data-total-target false JavaScript selector used to find the element to insert a running count of items.

Two Basic Examples

The examples below assume you are using Spring MVC/Boot for binding. If you are not then you may need to adapt this or use something else.

Spring divides lists into two and handles them differently. The first is @ElementCollection and the second is @OneToMany. Handling @ElementCollection lists is easier so we will start there.

Element Collection Example

The example below doesn't need to worry about index or sending notification to the server that an item needs to be deleted. You do need to let the server know that this list exists and if nothing is sent, the list needs to be cleared. For Spring, you use what is called a field marker prefix ('_' by default).

If you are copying an example, make sure your copying the right one. You most likely want the @OneToMany example below. One to Many Collection Example

One to Many Collection Example

For this example, you as the programmer will have to remove the item server side. The way this module expects you to do this is with a @Transient boolean on the entity. For the purpose of this example, we will use deleted as the name.

If the item has been saved to the database, we will need to add a hidden input to the DOM with the name deleted and a data attribute of data-delete-input. This tells the module that instead of removing the item from the DOM, we will instead hide it and set this input to true. Then server side we can find and delete any item where deleted is true.

It is easier to only include data-delete-input when there is an item saved to the database and you need to send a flag to remove it. There is no point to send a new item to the server, validate it, then delete it before you save it. With the hidden input missing, this module will just remove the item from the DOM and not send anything to the server.

We must also keep track of indexes in this type of collection. If you look at the template in the example below, you will find {index} in the template. This is replaced with the next index number when adding a new item to the list.

You might notice that this setup allows you to send items with gaps in the indexes. (e.g. Indexes 0, 1, 3 and, 6. Missing 2, 4, and 5.) This would normally create gaps in the collection server side. Spring handles this for us but if you are using a different backend you may want to pay attention to this.

Use inspection to check out how things work.

Note: Notice how the input with the data-delete-input attribute is not in the template. This is done on purpose. There is no point in sending those values to the server just to have the server remember to remove it from the list. Plus it would have to pass validation.

Inserting at the Top

You may want to insert new items at the top of the list instead of the bottom. You can specify that with the data-insert-location attribute on the template tag.

Make sure your indexes are also reversed (0 at the bottom and work your way up) so there are no surprises when you reload the page.
