How it works

The overall goal of these util functions is to show or hide something when the end user chooses some option. Another way to think about it is, 'If the user clicks on this radio/checkbox/select option, then that collapse (the target) will show or hide (the action).'

These utils use Bootstrap's Collapse class, so you must provide it as part of the initialization as the first parameter. The second parameter is optional and is a selector that is used to query the HTML document for all the radios, checkboxes, or selects that will work as controls for the collapses.

There are no options for these utils, they work off of two data attributes (both are required):

Name Description
data-target A selector that is used to find the Bootstrap collapse you want to manipulate.
data-action Can only be one of two options; show or hide. The action to take when the input/option is checked/selected. When unchecked/unselected, the opposite will happen.

Examples

Start by adding the initialization JavaScript to your page.

For simplicity in the examples below, all of the targets will be a simple div with an id, the collapse class, and the show class if defaulted to be seen. Just keep in mind that data-target is a selector, it can select classes or attributes as well. See Bootstrap's Collapse for more details.

Radio Buttons

Radio buttons are pretty straight forward, tag each one in the group with the action you want it to do. If you miss one, it may leave the collapse in a weird state. So make sure you get them all.

Checkboxes

Checkboxes work a little bit differently. The action is performed with the checkbox is checked, and the opposite action is performed on uncheck.

Open!!!

Things get a bit more complicated when multiple checkboxes work the same collapse. The line of thought that is supported is, 'if any of the checkboxes are checked then the collapse is shown', or 'if any of the checkboxes are checked then the collapse is hidden'. Don't mix actions that target the same collapse, it will produce unexpected results.

In the 'Good Example', if either checkbox is checked, the collapse is shown. If none of the checkboxes is checked, the collapse is hidden. This example is the HTML you can copy below the examples and everything works as expected.

The only difference in the 'Bad Example' is the action for the second checkbox is hide. I wanted to include it so others can play with it and see what happens. Try checking 'show checkbox', then 'hide checkbox', then uncheck 'show checkbox' and I think you will see.

Select Inputs

I have not tested a multi-select, and it is currently not supported.

There are currently two ways to configure a select input. The first is by putting the data-target attribute on the select directly and the data-action attribute on each option (including the empty option). This is nice for those selects with an 'other' option where you want to get more data when the end user selects other.

The other way is by putting the data-action attribute on the select and the data-target attribute on each of the options. This can create a kind of 'select your adventure' UI.
